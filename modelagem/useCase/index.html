    Introdução

    De acordo com definição do SEVOCAB, um Use Case é uma descrição das ações requisitadas de um sistema. Deste modo, este artefato busca, através da representação com um Diagrama de Caso de Uso, representar os Use Cases do sistema em questão: o Jitsi Meet. Todos os Use Cases criados foram baseados nos requisitos elicitados pelo grupo.

    Metodologia

    Para a representação dos Use Cases (descrição das ações requisitadas de um sistema, segundo o SEVOCAB), foi utilizado o Diagrama de Caso de Uso que, segundo definição do Lucidchart, visa apresentar de forma visual as possibilidades de interação do usuário com o sistema, apresentando também o escopo desse sistema.

    Um Diagrama de Caso de Uso é composto basicamente pelos Use Cases, pelos Atores, pelas Associações (que podem ser derivados em includes e extends) e pela Fronteira do Sistema. Sendo que, neste caso em específico, os Use Cases representam ações derivadas do que foi anteriormente requisitado na Elicitação de Requisitos.

    Componentes Comentários

    Atores


    		 São os usuários do sistema, podendo ser tanto primários (engajam o sistema) quanto secundários (reagem ao sistema).

    Casos de Usuário


    		 Representam os Casos de Uso que os atores podem realizar.

    Relacionamento de Associação


    		 Determinam um relacionamento com os atores que estão diretamente associados ao Caso de Uso.

    Relacionamento de Inclusão


    		 Determinam um relacionamento entre dois Casos de Uso em que um depende de outro.

    Relacionamento de Extensão


    		 Determinam uma relação entre dois Casos de Uso em que um não necessariamente depende do outro.

    Limites do Sistema


    		 Identifica o limite do sistema.
